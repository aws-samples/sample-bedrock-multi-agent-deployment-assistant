## Rules
1. Use EXACT AWS CloudFormation resource type names (e.g., "AWS::EC2::VPC").
2. Use EXACT AWS CloudFormation property names from the spec:
   - CidrBlock (not cidr_block), EnableDnsSupport (not enable_dns_support)
   - SecurityGroupIngress uses IpProtocol/FromPort/ToPort/CidrIp
   - EC2 Instance uses ImageId/InstanceType/SubnetId/SecurityGroupIds
   - NetworkInterface uses SubnetId/GroupSet (not SecurityGroupIds)
3. Apply the resolved deployment parameters for ALL specific values.
4. Resource naming convention: {{project}}-{{environment}}-{{resource-type}}-{{qualifier}}
5. Every taggable resource MUST have Tags: Project, Environment, ManagedBy ("cloudformation"), UseCase.
6. Logical IDs must be unique within your layer.
7. Compute instance specifics (if compute layer):
   - Create ENIs as separate AWS::EC2::NetworkInterface resources where needed
   - Attach ENIs via AWS::EC2::NetworkInterfaceAttachment
   - BlockDeviceMappings with Ebs.Encrypted: true
   - Include UserData with instance bootstrap configuration via {{"base64": {{"sub": "..."}}}}
8. Security groups (if security layer):
   - Management SG: SSH (22) and HTTPS (443) restricted to AdminCIDR parameter
   - Workload SG: Scoped to required ports and protocols for the deployment
