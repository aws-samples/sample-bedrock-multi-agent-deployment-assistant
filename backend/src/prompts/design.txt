## Topology Blueprint Rules

### VPC Topology
Each design option MUST include vpc_topology — a list of VPCs with:
- `role`: purpose from KB (e.g., "security", "inspection", "spoke", "management")
- `subnet_roles`: list of subnet types per AZ from KB (e.g., ["public", "private", "ha-sync", "ha-mgmt", "tgw-attach"])
- `availability_zones`: 1 or 2 (must match the resilience requirements)

### FortiGate Topology
Each design option MUST include fortigate_topology — a list of FortiGate instances with:
- `role`: instance role from KB (e.g., "active", "passive", "target", "hub")
- `vpc_role`: which VPC this FortiGate belongs to (must match a vpc_topology entry's role)
- `interfaces`: list of network interfaces, each with:
  - `port_name`: FortiGate port (e.g., "port1", "port2", "port3", "port4")
  - `subnet_role`: which subnet this port connects to (MUST exist in the VPC's subnet_roles)
  - `description`: purpose (e.g., "External/WAN", "Internal/LAN", "HA heartbeat", "HA management")

### Cross-Reference Validation
- Every interface `subnet_role` MUST exist in the corresponding VPC's `subnet_roles`
- Every FortiGate's `vpc_role` MUST match a VPC's `role`
- If HA mode is "active-passive" or "active-active", there MUST be at least 2 FortiGate instances
